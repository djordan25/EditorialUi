Principles

How EditorialUI thinks

Read this before adding a primitive, a variant, or a token. The goal is one coherent visual language an SR 11-7 examiner trusts on sight, that holds up across thousands of dense rows and hundreds of dialogs.

01

Reference posture

Bloomberg Terminal × Stripe Dashboard, with editorial restraint. Not Notion, not Atlassian, not consumer SaaS. Banks live in spreadsheets and reading rooms — the product should feel like an instrument paired with a quiet reading panel.

02

Generic primitives vs. domain wrappers

EditorialUI ships generic primitives only. Token-driven, single-responsibility, no Redux/routing/localStorage coupling. Domain wrappers (GraphLockSwitch, ProdicusGraphVersionSelect, ExaminerModeBanner) stay in ProdicusUI/ until phase 2 — they get rebuilt on top of EditorialUI primitives at consumer-migration time.

03

Selection components — when to use which

  • EdSelect≤7 fixed options, no search.
  • EdComboBox>7 options, in-list search.
  • EdAutocompleteFree-text input plus suggestions (tag input, search-as-you-type).
  • EdMenuCommand targets, not value selection.
  • EdContextMenuRight-click only.
  • EdFilterChipRowMulti-select filters above a table.
04

Tables vs. cards

EdDataTable (TanStack-backed, virtualized, sortable, selectable) is the going-forward primitive for inventory pages and dialog lists. EdNativeTable for static-display cases (3-row metadata grids in side panels). Cards are reserved for editorial moments and dashboard tiles, not list views.

05

Modals vs. dialogs vs. side panels

  • EdModalConvenience wrapper — title bar + button set + busy state. Default for forms and confirmations.
  • EdDialogLower-level Radix primitive. Reach for it when EdModal's API is too restrictive.
  • EdSidePanelInspect-without-leaving. Detail views, audit trails, run logs. Persistent or dismissable.
  • AvoidClick-away dismissal. Both Modal and Dialog suppress click-away by default; Escape always dismisses.
06

SR 11-7 severity

Findings, exceptions, and observations all share the five-step severity gradient: Low → Medium → High → MRA → MRIA. Don't pick ad-hoc colors per feature. Use EdStatusBadge with severity prop, or read --ed-color-severity-* tokens directly for SVG/chart code.

07

Audit-trail conventions

Superseded rows: --ed-color-row-superseded background, --ed-color-text-secondary text. Timestamps in --ed-font-mono for scannability. Never delete an audit-relevant row — supersede it. The replacing row references the superseded one's ID.

08

Examiner mode

When a user holds the EXAMINER role, a persistent banner renders between TopBar and main content. Read-only is enforced server-side; the banner is the user-visible reminder. EditorialUI primitives respect a global data-readonly attribute — disabled visuals applied without per-component handling.

09

Sentence case

Buttons, menu items, headings, dialog titles — all sentence case. Acronyms preserve their natural form (OK, MFA, MRA, SR 11-7). The text-transform property is not used for case conversion — the canonical label lives in the JSX.

10

Theme posture

Light is default for governance, auth, reading-room, and inventory surfaces. Dark is the long-session mode for graph builder and monitoring. Auth is policy-locked to light via data-theme="light" on the AuthLayout wrapper. To prevent FOUC, applyStoredThemePreference() runs at module load before React mounts.
