Decision rules

  • UsePage-level state that persists across renders — sync errors, mode banners (read-only, preview), SLA warnings, audit log notes.
  • PreferToast (out of scope here) for transient confirmations like "Saved". Notification is for state, not action receipts.
  • PreferEdEmptyState when the entire region has no data. EdNotification sits above content, not in place of it.
  • AvoidMore than 2 banners stacked. Group into a single banner that links to a list.
  • AvoidColor-only differentiation. Always include the icon.

A11y

  • Requiredrole="status" for info/success; role="alert" for warning/danger when the banner appears asynchronously. aria-live="polite" on info/success, "assertive" on danger.
  • RequiredDismiss button has aria-label="Dismiss notification". Esc dismisses when banner has focus-within.
  • RequiredColor contrast ≥ 4.5:1 in both themes. Severity icon is required because color isn't a sole cue.
