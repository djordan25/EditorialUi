Decision rules

  • UsePersistent metadata about an entity — what it IS (categories, regulations, version, area).
  • PreferEdStatusBadge for state (open, overdue, in-review). Different shape, different role.
  • PreferEdChip when the affordance is "toggleable filter" (pill shape, hover behavior).
  • AvoidTag clouds. Tags should be enumerable and meaningful, not decorative.

A11y

  • RequiredNon-interactive tag uses <span>. Don't apply role unless behavior changes.
  • RequiredClose button is a <button> with aria-label="Remove {tag name}".
  • RequiredTone color paired with label — never color alone (e.g. red tag without "deprecated" label).
