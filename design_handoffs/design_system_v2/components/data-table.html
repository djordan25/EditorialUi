Patterns

  • VirtualizationRequired for > 100 rows. Use TanStack Virtual. Row height fixed per density.
  • SortingSingle column at a time. Server-side for paginated tables (URL param). Client-side for fully-loaded sets.
  • SelectionCheckbox column on left. Header checkbox = select-all (page) or indeterminate when partial.
  • Bulk actionsAppear in toolbar when ≥ 1 row selected. Destructive (close, delete) gets danger styling.
  • PaginationFooter paginator with range + prev/next. Page size in column settings popover, persisted per user.
  • Row actionsTrailing icon-button opens an EdMenu. Don't put more than 2 inline icon-buttons per row.
  • Row clickOpens detail in EdSidePanel (or navigates to detail route). Checkbox click only toggles selection.

Decision rules

  • UseInventory pages: Findings, Models, Validators, Runs, Audit log. Any list with ≥ 3 columns or ≥ 50 rows.
  • PreferEdNativeTable for static 3-row metadata grids in side panels.
  • PreferEdList for picker results, recent items — anywhere selection is the point and rows have 1–2 fields.
  • AvoidCard grids in place of tables. Tables read as tables.

A11y

  • RequiredSemantic <table> + <thead> + <tbody> + <th scope="col">. Virtualizer must keep table semantics intact.
  • RequiredSortable headers: <button> inside <th aria-sort="ascending|descending|none">.
  • RequiredRow checkbox has aria-label="Select row F-2438". Selected row: aria-selected="true".
  • RequiredLive region announces selection changes: "3 of 312 selected".
  • RequiredKeyboard: Tab cycles to row checkbox + last action button. Arrow keys NOT used — tables are not grids in this product (Excel-style nav would conflict with screen readers).
