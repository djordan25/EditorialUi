Copy guidance

  • TitleVerb-led question for confirmations: "Reopen finding F-2438?". Statement for forms: "New finding".
  • BodyWhat will happen, in past tense. "Closure timestamp will be cleared and the finding will re-enter the queue."
  • Primary actionEcho the title verb: "Reopen", "Delete period", "Open finding". Never "OK" or "Submit".
  • CancelAlways "Cancel" for confirmations. "Keep editing" when interrupting unsaved work.

Decision rules

  • UseConfirmations, focused forms (≤ 8 fields), wizards, busy interruptions.
  • PreferEdDrawer when the content is about a specific row/entity in the current list — drawer keeps context.
  • PreferInline forms when the form lives within a page workflow.
  • AvoidModals as the default UI for everything. They block context and interrupt scanning.

A11y

  • RequiredInherits all of EdDialog's a11y. Adds: primary action receives default focus when no input is present; first input when one is.
  • RequiredBusy state sets aria-busy="true" on dialog and disables non-cancel actions. Live region announces stage changes ("Stage 2 of 4: validating columns").
