Decision rules

  • UseSettings that flip a real-world state instantly (notifications on, dark mode on, examiner-mode banner visible).
  • UseInline within a row of settings, flush right.
  • AvoidInside forms with a Save button. If the change requires confirmation, use EdCheckbox.
  • AvoidGrouping multiple switches as a multi-select. Use EdCheckbox for selection semantics.

A11y

  • RequiredUse Radix Switch — provides role="switch" + aria-checked + keyboard support.
  • RequiredLabel is part of the click target. Don't separate switch from its label.
  • RequiredIf the change is async, optimistically toggle UI but show a "saving…" inline; revert + show error on failure.
