Trigger — right-click on a target row, card, or node. Long-press on touch.
Target highlight — target row gets a 2px brand outline while menu is open.
Anchor — menu appears at click position, auto-flips if it would clip viewport.
Items — same look as EdMenu. Don't invent new visual patterns.

Triggers

table row
Right-click anywhere in the row body. Reveals row-scoped actions: open, reassign, status changes, copy ID, delete.
graph node
Right-click on a node in the dependency graph. Reveals node-scoped actions: zoom to, expand subtree, hide, copy node ID.
canvas background
Right-click on empty canvas. Reveals canvas-scoped actions: fit to view, reset zoom, paste.
text selection
Right-click within a code/log block with selection. Reveals copy / search / open. Defer to native browser menu when unsure.

Behavior

  • RequiredBuild on Radix ContextMenu. Suppresses native browser menu when the trigger area is registered.
  • RequiredOn open, target gets a visible focus / selection outline. Removed on dismiss.
  • RequiredItems must mirror an action available in the visible UI (row toolbar, edit button). Context menus are accelerators, never the only path.
  • RequiredLong-press (700ms) on touch triggers the menu. Visual press feedback during the hold.
  • AvoidHiding destructive actions only in the context menu. Delete must be reachable from the visible row.

Decision rules

  • UsePower-user accelerators on table rows, graph nodes, and canvases.
  • PreferEdMenu for declared, button-triggered actions.
  • AvoidContext menus inside dialogs and modals. Too many input modes.

A11y

  • RequiredKeyboard equivalent: focused row + Shift+F10 or Menu key opens the same menu at the row's bounding box.
  • RequiredMenu uses Radix's full a11y wiring — role=menu, menuitem, focus management.
  • RequiredTouch trigger announces "Press and hold for actions" on first encounter.
