Decision rules

  • UseTop-of-table fast filters, faceted lists, status filters where users want a one-click toggle.
  • Use3–8 chips visible. Beyond that, collapse extras into "+N more".
  • PreferEdComboBox (multi mode) when the value space is large or unfamiliar.
  • PreferEdSegmentedControl when there are exactly 2–4 mutually-exclusive view modes (canonical control).
  • AvoidMore than one ChipRow stacked. Use a divider in a single row, or restructure into a filter drawer.
  • AvoidCounts that change without user-visible feedback (e.g. cross-filter math). If counts will mislead, omit them.

A11y

  • RequiredMulti-select: each chip is a <button role="checkbox" aria-checked>. Group has role="group" aria-label="Filters".
  • RequiredSingle-select: role="radiogroup"; chips are role="radio" aria-checked. Arrow keys move focus.
  • RequiredCounts read as "Open, 87 items". Use aria-label on chip to combine label + count.
  • RequiredDon't rely on color alone — selected state has both bg AND border change vs default.
