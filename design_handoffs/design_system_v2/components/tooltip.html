Behavior

  • RequiredOpen delay 500ms on first hover; instant on subsequent hovers within 500ms (group hover).
  • RequiredClose immediately on mouseleave; close on Esc when keyboard-shown.
  • RequiredOpen instantly on focus (no delay) for keyboard users.
  • RequiredAuto-flip placement when bubble would clip viewport. Use Floating UI / Radix flip middleware.
  • AvoidTooltips on touch — they don't trigger reliably. Replace with a popover or render the label inline at touch breakpoints.

Decision rules

  • UseDisambiguate icon-only controls. Show keyboard shortcuts. Reveal full text behind ellipsis. Define jargon in lists/tables.
  • PreferEdPopover for content that needs interaction (links, copy buttons, multi-line forms). Tooltip is for read-only hints.
  • AvoidCritical info in tooltips — keyboard, touch, and screen-reader users may miss them. If it's required, put it in the page.

A11y

  • RequiredBuild on Radix Tooltip — provides correct aria-describedby wiring and focus visibility.
  • RequiredThe trigger is focusable. Tooltip has role="tooltip", never receives focus itself.
  • RequiredFor icon-only buttons, the tooltip text duplicates aria-label — both must match.
  • RequiredHonor prefers-reduced-motion: disable the open/close fade.
