Foundations

Tokens, type, and rhythm

EditorialUI is the design language for ModelRiskIQ. Restrained, dense, typographic — built for regulated finance, not consumer apps. Every primitive consumes the tokens on this page; nothing else. If a value isn't here, add it here first.

01

Posture

Bloomberg Terminal × Stripe Dashboard, with editorial restraint. Information lives in tables and reading panels, not cards. Color is reserved for meaning. Display sizes earn their slot. Sentence case throughout.

  • UseCool-grey neutrals, generous hairlines, a single cobalt accent. Type does the heavy lifting.
  • UseLight by default for governance, auth, and reading-room surfaces. Dark for long-session work — graph builder, monitoring.
  • PreferTables to cards. Tables read as tables. Cards are reserved for editorial moments and dashboards.
  • AvoidDecorative gradients, drop shadows on flat surfaces, emoji, color used for branding rather than meaning.
  • AvoidTitle Case. Sentence case for buttons, headers, labels. Acronyms keep their natural form (OK, MFA, MRA, SR 11-7).

07

Motion

Motion is functional, never decorative. Hover any card below to see the curve. All durations honor prefers-reduced-motion — they collapse to 0ms.

transition-fast

80ms ease · button press, focus shift

transition-base

120ms ease · default · most state changes

transition-slow

200ms ease · drawer, modal open

transition-spring

320ms cubic-bezier · cross-fades, scene changes

08

Iconography

Single icon family — line-weight 1.5px on a 16px grid. Do not mix families. EdIcon wraps the source, exposes size (xs/sm/md/lg) and color (semantic — inherit, primary, brand, success, warning, danger).

Icon family is not yet selected. Recommendation: lucide-react (line-weight, comprehensive, MIT, swap-compatible with the existing react-icons). Implementer to confirm before phase 1B finishes.

Implementer notes

EditorialUI ships in parallel with the existing ProdicusUI library. Don't replace; co-exist until consumers are migrated.

  • LandTokens go in src/theme/tokens-v2.scss. Lift shared/tokens.css verbatim — token names are already prefixed --ed-* so they don't collide.
  • LandJS access via src/theme/tokens-v2.ts, mirroring the existing tokens.ts shape: cssVar('color-brand'), token('color-brand'), plus typed token unions.
  • LandComponents live under src/components/EditorialUI/. Component prefix Ed (e.g. EdButton, EdDataTable). One folder per component, one Storybook story file alongside.
  • UseStorybook category EditorialUI/<Group> — Inputs, Selection, Feedback, Containers, Display, Navigation, Data. The category mirrors this spec doc's grouping.
  • UseTheme switching: same data-theme attribute as the existing system — one mechanism, both libraries respect it.
  • Don'tDon't wrap a Radix primitive in EditorialUI if a thin adapter over the existing ProdicusUI primitive will do — but only if the existing primitive is genuinely token-driven and stateless. Most aren't; build fresh on Radix.
  • Don'tDon't add a new token mid-implementation without updating this doc and tokens.css. The doc is the contract.
