01 Components

G1 · Data

EdDataTable

Generic, column-config driven inventory table. Per-column sort (client or server), row selection with toolbar + bulk actions, superseded rows, row actions, pagination, density, and empty / loading / error states. Semantic table throughout; virtualize at the call site for >100 rows.

G2 · Data

EdNativeTable + EdKeyValueTable

Static plain tables for small known-row sets — no sort/select/virtualization. EdKeyValueTable is the label→value grid (th scope=row) for side panels; EdNativeTable is the columned variant with zebra + custom renderers. Reach for these instead of hand-rolling a table.

F3 · Data

EdList

Vertical list of selectable / scannable rows — picker results, recent items, entity selectors. Static ul or single-select listbox with arrow-key nav. Leading avatar/icon, title + description, trailing meta (folded into the accessible name), groups, empty + skeleton states.

04 Dependencies & integration

  • — none new —Plain React + CSS + lucide. Virtualization (TanStack Virtual) is an opt-in call-site wrap for >100 rows.
  • Client vs serverOmit sort/page props for client-side; pass them (+ onSortChange/onPageChange) for server-paginated tables.
  • Native checkboxSelection uses a styled native checkbox to stay dep-free; swap in EdCheckbox (B2) if you want that exact visual.

05 What this codifies

  • Tables read as tablesSemantic table/thead/th scope throughout. Never a card grid in place of a table.
  • Presentation, not dataThe table owns sort/select/paginate UI + ARIA; fetching, server-state, and virtualization stay at the call site (TanStack).
  • Column-config, not childrenEdColumn<Row>[] drives the table — map your data, don't nest <Column> elements.
  • Checkbox ≠ row-clickSelecting a row never triggers onRowClick; the checkbox cell stops propagation.
  • Static gets a primitive tooEven 3-row metadata blocks use EdKeyValueTable so they inherit type / spacing / dark-mode.
