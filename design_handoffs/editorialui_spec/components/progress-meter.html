Decision rules

  • UsePredictable, measurable jobs — uploads, imports, sync, batch processing.
  • UseSegmented variant when work has discrete stages with names. Pair with a stage strip.
  • PreferEdCircularProgress for spinners and cell-level loads.
  • PreferEdSkeleton when you can show the shape of arriving content.
  • AvoidShowing % when you have to invent it. Indeterminate is honest; fake progress is hostile.

A11y

  • Requiredrole="progressbar" + aria-valuenow / aria-valuemin / aria-valuemax / aria-valuetext. aria-valuetext reads the visible string ("68%, 2,142 of 3,150").
  • RequiredIndeterminate: omit aria-valuenow. Use aria-busy="true" on the host region.
  • RequiredLive updates wrap in aria-live="polite" region announcing milestones (25/50/75/100), not every tick.
  • RequiredHonor prefers-reduced-motion: disable the indeterminate sweep, fall back to "Working…" label.
