EdButton

Primary action target. Six variants for six intents — the variant is a semantic choice, not a styling one. Resting height aligns with all other controls so toolbars read as a single line.

01

Variants

One primary per surface. Secondary for paired actions. Ghost for tertiary / inline. Danger for destructive. Link for inline text actions.

07

Decision rules

  • Use primaryFor the single most-likely action on a surface. One per dialog footer; one per command bar.
  • Use secondaryPaired with primary (e.g. Cancel + Save). Toolbar buttons that aren't the primary action.
  • Use ghostTertiary actions that should fade until hovered (table row inline actions, breadcrumb back, header utilities).
  • Use dangerOnly for destructive, non-reversible actions. Always confirm in modal first.
  • AvoidStacking 3+ primary buttons on one surface. If the page has many actions, use one primary + an EdMenu for the rest.
  • AvoidUsing danger as a primary just to make it pop. Color is for meaning, not emphasis.
08

A11y

  • RequiredNative <button type="button">. Submit-style buttons inside forms use type="submit".
  • RequiredLoading state sets aria-busy="true" + aria-disabled="true". Don't strip the label — keep it for screen readers; visually hide via opacity 0.
  • RequiredFocus ring visible at all times when focused (focus-visible). Never outline: none without a visible replacement.
  • RequiredIcon-only buttons go through EdIconButton (which forces aria-label). Never strip the label from EdButton to make it icon-only.
