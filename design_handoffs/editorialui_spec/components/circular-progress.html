When NOT to use a spinner

  • Prefer skeletonsFor content that loads with predictable structure — table rows, card grids, drawer details. Use EdSkeleton.
  • Prefer progress meterFor long, predictable jobs (file upload, batch import). Use EdProgressMeter with stages.
  • Spinner is correctOperations > 200ms but with no progress signal: form submit, search query, sync trigger, cell-level loads.

A11y

  • Requiredrole="status" + aria-live="polite" + visually-hidden label e.g. "Loading findings".
  • RequiredDeterminate ring uses role="progressbar" + aria-valuenow / aria-valuemin / aria-valuemax.
  • RequiredHonor prefers-reduced-motion: replace rotation with a pulse/stripe animation at 1.5s.
