First-time tour

Walk through the finding lifecycle in 4 steps. Takes 90 seconds.

Behavior

  • RequiredBuild on Radix Dialog. Focus traps within content; focus returns to trigger on close.
  • RequiredEsc dismisses. Overlay click dismisses by default; preventOutsideClose prop disables for unsaved-form guards.
  • RequiredAuto-focuses first focusable in body, OR an element with autoFocus. Title gets initial focus only if no input is present.
  • RequiredBody scrolls when content exceeds height. Header + footer remain pinned.
  • AvoidStacking dialogs. Don't open a second dialog from inside one — use a confirmation step inline, or replace the content.

Decision rules

  • UseBuilding blocks for EdModal, EdConfirmation, multi-step forms.
  • PreferEdModal for the 95% case — it preconfigures sensible defaults.
  • PreferEdDrawer when content is contextually about a row/entity. Drawer keeps the page navigable.
  • AvoidDialogs for transient confirmations like "Saved". Use a toast.

A11y

  • Requiredrole="dialog" + aria-modal="true" + aria-labelledby pointing at the title.
  • RequiredBody content gets aria-describedby when there's a clear description paragraph.
  • RequiredOutside-of-dialog content is aria-hidden="true" while open. Radix handles automatically.
