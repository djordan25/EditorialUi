Date formatting

  • RequiredISO format YYYY-MM-DD in trigger and inputs. Mono font.
  • RequiredRange separator is → (rightwards arrow) with one space each side. Never a hyphen.
  • RequiredWhen a preset matches exactly, show preset name in trigger ("Last 14 days") instead of literal dates.
  • AvoidLocale-formatted dates in this product. Audit/regulatory work demands unambiguous ISO.

Presets — recommended set

const RANGE_PRESETS = [
  { label: 'Today',          range: () => [today(), today()] },
  { label: 'Yesterday',      range: () => [yesterday(), yesterday()] },
  { label: 'Last 7 days',    range: () => [subDays(today(), 6), today()] },
  { label: 'Last 14 days',   range: () => [subDays(today(), 13), today()] },
  { label: 'Last 30 days',   range: () => [subDays(today(), 29), today()] },
  { label: 'This quarter',   range: () => [startOfQuarter(today()), today()] },
  { label: 'Last quarter',   range: () => [startOfQuarter(subQuarters(today(), 1)), endOfQuarter(subQuarters(today(), 1))] },
  { label: 'Year to date',   range: () => [startOfYear(today()), today()] },
];

Decision rules

  • UseDate-range filters in tables, audit period selection, finding age windows, exam date pickers.
  • UseSingle-date variant for "Due date" or "Effective date" inputs (one calendar, no presets).
  • PreferPresets-only (no calendar) when the choice is always a relative window — use EdSelect.
  • AvoidTime-of-day in this product. We don't model hours/minutes for findings or exams.

A11y

  • RequiredBuild on react-aria DateRangePicker or Radix Popover + custom calendar grid. role="grid" on calendar.
  • RequiredArrow keys move focus by day; PageUp/Down by month; Home/End to row edges.
  • RequiredRange announcement: "Selected March 2 to March 15, 2026 — 14 days."
  • RequiredPresets are buttons in a list; selected preset has aria-pressed="true".
