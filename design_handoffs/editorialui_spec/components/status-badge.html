Decision rules

  • UseSoft for the workhorse 95% of cells. Solid for MRA/MRIA and other escalation-level signals. Outline for archived or read-only context.
  • UseThe status dot only when the state implies motion or recent change (Open, In Review). Skip on terminal states (Closed, Resolved).
  • PreferEdTag for non-status metadata (categories, regulations, business unit). Sentence case, lower visual weight.
  • AvoidInventing new tones per page. Pick from {neutral, info, success, warning, danger, brand} and use the canonical map.

A11y

  • RequiredColor is paired with a label — never a bare colored dot. Soft variants meet 4.5:1 against their backgrounds.
  • RequiredIf badge replaces text in a row, add aria-label with the full state name ("Status: Overdue").
  • RequiredBadge is non-interactive. If users can change status, wrap the row or render a separate menu — don't attach onClick to the badge.
