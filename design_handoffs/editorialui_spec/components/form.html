Patterns

  • LayoutSingle-column default. 2-column rows for paired inputs that read as a unit (severity + date, country + region).
  • ValidationValidate on blur after first interaction, then on every keystroke once a field has errored. Show inline error AND aggregate summary when more than one field fails.
  • SubmitDisable primary button only when validation has failed AND the user has tried to submit. Don't gate on initial state.
  • AutosaveFor drafts, autosave every 5s on idle. Show "Saved 11:42" inline; reuse aria-live="polite".
  • GuardUnsaved changes trigger a sticky bar at the bottom AND a confirmation on route change.

Decision rules

  • UseAny form with 2+ fields. Wraps react-hook-form's FormProvider with our layout primitives.
  • PreferEdModal wrapping EdForm for focused, ≤ 8-field forms.
  • PreferA full route for forms > 12 fields or multi-step wizards. Forms-in-modals start to feel claustrophobic.
  • AvoidInline-editable rows in tables for anything beyond a single field. Use a side-panel form.

A11y

  • RequiredWrapping <form>. Submit button is type=submit. Cancel/discard are type=button.
  • RequiredGroup titles use heading element appropriate to nesting (h3 inside a modal, h2 on a page).
  • RequiredError summary banner at top is role="alert". Each inline error has aria-describedby wired from field.
  • RequiredFocus moves to first errored field on submit failure.
