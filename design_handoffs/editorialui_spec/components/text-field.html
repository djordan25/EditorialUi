Decision rules

  • UseFor any single-line string input. Required indicator goes on the label, not as helper text.
  • UseMono variant for IDs, IPs, paths, codes, regex, JSON values inside a single field.
  • PreferHelper text over placeholder for the kind of value expected. Placeholder vanishes on type — helper persists.
  • AvoidInline labels (label inside the field). Eyebrow labels read better in dense forms.
  • AvoidShowing both hint and error simultaneously. Error replaces hint.

A11y

  • RequiredLabel is always rendered. Use visuallyHidden only when label is duplicated semantically (e.g. search input next to a "Search" eyebrow).
  • RequiredError sets aria-invalid="true". Hint/error linked via aria-describedby.
  • RequiredRequired state sets aria-required="true"; the * on the label is decorative.
