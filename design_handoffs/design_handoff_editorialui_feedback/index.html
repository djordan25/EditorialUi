01 Components

C5 · Feedback · Radix Tooltip

EdTooltip

Anchored, transient label on hover / focus. Simple (with optional kbd chip) or rich (title + body). Dark-on-light, auto-flips, honors reduced-motion. Wrap the app once in EdTooltipProvider for instant grouped hovers. Read-only hints only.

C1 · Feedback

EdNotification

Inline banner for persistent page / section state. Four severities (each with a required icon — never color-only), three variants (rail / subtle / compact). ARIA role derived from severity. Actions are ghost/secondary, never primary. Not a toast; not a replacement for content.

tsx EdNotification.tsx scss EdNotification.module.scss stories EdNotification.stories.tsx tests EdNotification.test.tsx

D1 · Containers · Radix Dialog

EdDialog + EdConfirmation

Radix modal surface — overlay, focus trap, header / body / footer. Four sizes, danger styling, preventOutsideClose for unsaved-form guards, scrolling body. EdConfirmation is the preset for the destructive / confirm case — pass renderActions with your EdButtons.

tsx EdDialog.tsx (+ EdConfirmation) scss EdDialog.module.scss stories EdDialog.stories.tsx tests EdDialog.test.tsx

C2 · Feedback

EdEmptyState

Centered title + body + actions for a region with no data. Tones (neutral / danger / success), restrained single-glyph visual in a dashed ring — no illustrations. compact for small panels, headingLevel matches the document outline. Institutional copy tone.

05 What this codifies

  • Color is never the only signalEdNotification requires an icon per severity; empty-state tones carry icon + text. No color-only differentiation.
  • ARIA role is derivedEdNotification picks status vs alert from severity. role="none" opts out for load-present banners.
  • Banner ≠ CTANotification actions are ghost / secondary only. A primary button in a banner is a smell — it's probably an empty state or a dialog.
  • Confirm is state-drivenEdConfirmation replaces window.confirm — non-blocking, wire onConfirm rather than awaiting a boolean.
  • Empty state has institutional tonePlain title, cause + unblock body, primary-verb actions. No mascots, no "Oops!", no animated illustrations.
